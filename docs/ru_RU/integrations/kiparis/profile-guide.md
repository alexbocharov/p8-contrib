# 🛸 KIPARIS: Руководство по созданию профиля

**Профиль интеграции** — это XML-файл, определяющий «паспорт» внешней системы. Он диктует Парусу, какие поля отрисовать, как их валидировать и каким способом защищать данные.

## 🏗 Структура XML

### 1. Заголовок системы (`<system>`)

* **`code`**: Уникальный технический идентификатор (напр. `DADATA`). До 30 символов.
* **`name`**: Понятное название системы для пользователя.

### 2. Описание полей (`<field>`)

Каждое поле — это строка в будущих настройках. Атрибуты определяют его поведение.

**Основные атрибуты:**

* **`fld_code`**: Технический код параметра (напр. `api_url`).
* **`fld_name`**: Текст подсказки/заголовка в интерфейсе (до 2000 симв.).
* **`sort_order`**: Порядок отображения (рекомендуемый шаг — 10).

**Типизация и интерфейс:**

* `data_type` (физический тип данных):
   * `0` — Строка (STR)
   * `1` — Число (NUM)
   * `2` — Дата (DATE)
* `entry_type` (способ формирования):
   * `0` — **Ручной ввод** (обычное поле).
   * `1` — **Секретный ввод** (маскировка `***`, шифрование **AES-256**).
   * `2` — **Чекбокс (Да/Нет)**. Требует `data_type="1"`.
   * `3` — **Контейнер (Bundle)**. Невидимое поле-приемник для секретов.

**Безопасность (stpolicy):**

* `0` — **Local:** Хранится в БД Паруса как есть.
* `1` — **Vault:** Только внешнее хранилище (в БД остается ссылка).
* `2` — **Hybrid:** Шифруется **AES-256** в БД Паруса, но может быть передано в Vault.

## 🔐 Работа с группами секретов (Bundling)

Если вы хотите объединить несколько полей (напр. логин и пароль) в один защищенный пакет в Vault:

   1. **Поля-доноры:** Укажите у них атрибут `bundle="имя_контейнера"`.
   2. **Поле-контейнер:** Создайте поле с `fld_code="имя_контейнера"` и установите `entry_type="3"`.

> Важно: Поля с `entry_type="3"` автоматически скрываются при создании новой настройки и появляются только после активации режима защиты.

## 🛠 Пример быстрого старта

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kipintmanifest version="1.0">
  <system code="DADATA" name="DaData.ru">
    <fields>
      <field fld_code="api_token" 
             fld_name="API Токен" 
             sort_order="10" 
             entry_type="1" 
             data_type="0" 
             stpolicy="2" />
    </fields>
  </system>
</kipintmanifest>
```

*Этот код создаст в Парусе скрытое поле, данные которого будут зашифрованы AES-256 прямо в базе.*

## 🚀 Загрузка в систему

   1. Зайдите в раздел **«Виды интеграций»**.
   2. Выполните действие **«Загрузить профиль»**.
   3. Выберите ваш XML-файл.
   4. Система автоматически обновит метаданные, не затронув существующие подключения клиентов.
